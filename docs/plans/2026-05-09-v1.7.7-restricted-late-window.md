# v1.7.7 — Restricted Late-Phase Window for Goal-Stack Hypothesis Test

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Test whether the goal-stack mechanism produces a ≥10pp drop in late-phase trap-hit-rate when the late-phase metric is narrowed from "last third (7 traps)" to "last 4 trap encounters". v1.7.6 confirmed budget reduction does not discriminate; this plan acts on the pre-committed v1.7.7 escalation. Verdict is DEMONSTRATED, NOT_SUPPORTED, or SANITY_FAIL — published with measured numbers + CIs in every case.

**Architecture:** Three logical pieces, all built on v1.7.5/v1.7.6 infrastructure (do not rebuild). (1) **Workload knob** — `--restrict-late-to <int>` flag on `run.mjs`. When set, late slice = last N trap encounters; early/mid re-split to "first ceil((total-N)/2)" / "remainder" so the three slices stay disjoint and total to all trap tasks. Default `null` preserves current chronological-third behavior. `buildOutput()` records the value in result JSON for audit. (2) **C2 sanity preflight** — same C2-before-C3 design from v1.7.6 P0-2: run C2 only at `--budget 2000 --restrict-late-to 4 --n-seeds 20 --eval-strict`, mechanically check C2 late mean ∈ [0.04, 0.24] AND lower-CI > 0; on FAIL, STOP, do NOT collect C3, retarget to v1.8 adversarial categories. (3) **Hypothesis run + decision** — only if C2 sanity passes. Run C0, C1, C3 at the same workload; analyzer emits verdict (SUPPORTED / NOT_SUPPORTED / SANITY_FAIL / HOOK_FAIL) per pre-registered rule. All three verdict templates committed BEFORE the run.

**Tech Stack:** Node 22.5+ (built-in `node:sqlite`), vitest, the v1.7.5 multi-seed harness (`run.mjs`, `aggregate.mjs`, `traps.mjs`), `pairedPermutationCI` (10k Monte Carlo resamples), `hippo` CLI subprocess via `execSync`. **Step 3: Update `hitRateByPhase` signature in run.mjs**

```javascript
/**
 * Split trap results into phases and compute hit rate for each.
 *
 * v1.7.7: when `restrictLateTo` is null (default), late = last third of traps
 * (chronological third). When `restrictLateTo = N` (positive integer), late =
 * last N traps; early/mid re-split to "first ceil((total-N)/2)" / "remainder"
 * so the three slices stay disjoint and exhaustive.
 *
 * @param {Array<{trapCategory: string|null, trapHit: boolean}>} results
 * @param {number|null} [restrictLateTo=null] - Optional override for late slice size.
 * @returns {{early: number, mid: number, late: number}}
 */
export function hitRateByPhase(results, restrictLateTo = null) {
  const trapTasks = results.filter((r) => r.trapCategory !== null);
  const n = trapTasks.length;

  const rate = (slice) => {
    if (slice.length === 0) return 0;
    return slice.filter((r) => r.trapHit).length / slice.length;
  };

  if (restrictLateTo === null) {
    // Default: chronological third (v1.7.0..v1.7.6 behavior).
    const third = Math.ceil(n / 3);
    return {
      early: rate(trapTasks.slice(0, third)),
      mid: rate(trapTasks.slice(third, third * 2)),
      late: rate(trapTasks.slice(third * 2)),
    };
  }

  // v1.7.7: late = last N. early = first ceil((n-N)/2). mid = remainder.
  const N = Math.max(0, Math.min(restrictLateTo, n));
  const earlyEnd = Math.ceil((n - N) / 2);
  const midEnd = n - N;
  return {
    early: rate(trapTasks.slice(0, earlyEnd)),
    mid: rate(trapTasks.slice(earlyEnd, midEnd)),
    late: rate(trapTasks.slice(midEnd)),
  };
}
```

**Step 4: Add `parseRestrictLateTo` helper + export**

Add near `parseArgs`:

```javascript
/**
 * Parse and validate the --restrict-late-to flag value. Returns null if input
 * is the literal string 'null' (programmatic CLI authors). Otherwise expects a
 * non-negative integer; throws otherwise.
 *
 * @param {string} raw
 * @returns {number}
 */
export function parseRestrictLateTo(raw) {
  if (typeof raw !== 'string') {
    throw new Error(`--restrict-late-to expects a string; got ${typeof raw}`);
  }
  if (!/^\d+$/.test(raw)) {
    throw new Error(`--restrict-late-to expects a non-negative integer; got "${raw}"`);
  }
  return parseInt(raw, 10);
}
```

```javascript
/**
 * Pure verdict computation. No I/O. Testable in isolation.
 * @param {{c2Late:number, c3Late:number, delta:number, ciLow:number, ciHigh:number,
 *          hookFailures:{push:number, complete:number}|null,
 *          sanityPass:boolean}} input
 * @returns {'SUPPORTED' | 'NOT_SUPPORTED' | 'SANITY_FAIL' | 'HOOK_FAIL'}

export function computeVerdict({ c2Late, c3Late, delta, ciLow, ciHigh, hookFailures, sanityPass }) {
  if (!sanityPass) return 'SANITY_FAIL';

  if (delta >= 0.10 && ciLow > 0) return 'SUPPORTED';

}
```
